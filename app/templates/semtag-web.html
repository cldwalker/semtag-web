First time here? Learn more about semtag.

Semtag is a site that stores bookmarks as entities. Each entity must have a type, similar to a category. Unlike a category, a type is meant to specifically model how we view the world and answer "What is this thing? Is it an article, app, song, clip, etc. ?". For example, here is my bookmark for google which is a company type. The types can be any value and thus allows you to model your knowledge. For example, these are my types.

Semtag uses tagging as a means of organizing bookmarks, much like previous bookmarking sites e.g. delicious.com. With these older sites, tags mainly served to create a list of things e.g. google things and little importance was given to the tag itself. In semtag, tags are as important as the entities they are tagging; they are entities themselves. This means that tags are not just creating lists, they are creating relationships between entities. For example, google is an entity that has this related (tagged) list of articles, products, libraries, etc.

By treating bookmarks as entities, letting you create any types for them and freely associate between them, semtag provides a powerful way to organize and grow your taxonomy.
